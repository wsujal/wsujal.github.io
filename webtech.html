WWW & The Internet PYQ 3M

WWW (World Wide Web) is a global information medium — a system of interlinked hypertext documents accessed via browsers using HTTP. Tim Berners-Lee proposed the concept in 1989 while at CERN, and it expanded into the modern Web by 1990.

Internet

  • Global network of networks
  • Infrastructure layer — TCP/IP
  • Carries email, FTP, WWW & more
  • Connects millions of computers

WWW

  • One service running on the Internet
  • Uses HTTP protocol
  • Accessed via browsers (Chrome, Firefox)
  • Documents linked by hyperlinks & URLs
Key Distinction: Internet ≠ WWW. The Internet is the physical infrastructure (cables, routers, protocols). WWW is software — web pages you access via a browser using HTTP.

HTTP Protocol PYQ 3M PYQ 4M

HTTP (Hypertext Transfer Protocol) is a stateless, request-response protocol running over TCP/IP. Stateless = the server doesn't remember previous requests. Each request is independent.

HTTP Header Types

General Header

For both request & response. e.g. Date, Connection

Request Header

Only in requests. e.g. Host, Accept, User-Agent

Response Header

Only in responses. e.g. Server, Set-Cookie

Entity Header

Describes the body. e.g. Content-Type, Content-Length

Web Design Issues PYQ 4M PYQ 7M

🌐 Browser & OS Compatibility▼

Different browsers interpret HTML/CSS differently. Same browser can behave differently across OS platforms. Solution: Test on multiple browsers and operating systems before releasing.

📶 Bandwidth & Cache▼

Users have different speeds. Too many images on a low-bandwidth connection = slow load. Users won't wait more than 10–15 seconds. Browsers use cache (temporary memory) to store graphics so repeat visits are faster.

🖥️ Display Resolution▼

Screen resolution varies (800×600, 1024×768, 1920×1080+). Three design approaches: Fixed layout, Flexible layout using CSS/tables, or Centered design. Best practice: use responsive frameworks like Bootstrap.

🧭 Navigation & Effective Tips PYQ 3M▼
  • Links should be clear, meaningful, and consistent
  • Group content logically — don't dump everything in one nav
  • Provide a way to return to the home page from every page
  • Use horizontal nav bars for quick section jumping
  • Show users where they are (breadcrumbs, active link highlight)
  • Include: About Us, Contact Us, Search
🎨 Look, Feel & Page Layout▼

Covers website theme, typography, graphics, and visual structure. Screen is divided into 5 priority areas: center → top → right → bottom → left. Design should be user-centric — users scan first, read later.

HTML Frames PYQ 7M

Frames divide the browser window into sections, each loading a separate HTML page. Deprecated in HTML5 — use CSS Flexbox/Grid instead. Still asked in exams — know the syntax.

<!-- Vertical split: 30% nav | 70% content -->
<frameset cols="30%,70%">
  <frame src="menu.html"   name="menu">
  <frame src="content.html" name="content">
</frameset>

<!-- Horizontal split then vertical -->
<frameset rows="80px,*">
  <frame src="header.html">
  <frameset cols="200px,*">
    <frame src="sidebar.html">
    <frame src="main.html">
  </frameset>
</frameset>
Exam Note: Frames are removed in HTML5 but still appear in PYQs. Always mention they are deprecated and suggest CSS layouts as the modern alternative.

CSS — Types & Syntax PYQ 7M

CSS (Cascading Style Sheets) defines how HTML elements are displayed. Rules cascade — later/more-specific rules override earlier/less-specific ones.

CSS Syntax

selector {
  property: value;    /* one declaration */
  property: value;
}

3 Ways to Write CSS PYQ 7M

1. Inline CSS — Highest Priority▼

Written inside element's style attribute. Hardest to maintain, overrides everything else.

<p style="color: red; font-size: 18px;">Red paragraph</p>
<h1 style="background: blue; color: white;">Blue Heading</h1>
2. Internal / Embedded CSS — Single Page▼

Written inside <style> tag in <head>. Good for single-page styles only.

<head>
  <style>
    body { background-color: black; }
    p    { color: white; font-size: 16px; }
    h1   { color: orange; }
  </style>
</head>
3. External CSS — Best Practice ✓▼

Separate .css file linked via <link>. One file styles the whole website — change once, updates everywhere.

<!-- In HTML <head> -->
<link rel="stylesheet" type="text/css" href="style.css">

/* style.css */
body { background-color: #f0f0f0; }
p    { color: #333; line-height: 1.6; }
h1   { color: #e8673a; }
Priority Order: Inline > Internal > External > Browser Default

  • Higher specificity than class

JavaScript Introduction PYQ 3M

JavaScript is a lightweight, interpreted scripting language that runs in the browser. It makes pages dynamic — responding to user actions, updating content without reloading.

Client-Side JS

  • Runs in the browser
  • Uses browser resources, not server
  • Saves bandwidth
  • Code is visible to users
  • Can be disabled by user

Server-Side JS

  • Runs on web server (Node.js)
  • Can access databases directly
  • Logic is hidden from user
  • Generates dynamic HTML responses
  • Cannot be disabled by user

Prime Number Check PYQ 7M

function isPrime(n) {
  if (n < 2) return false;
  for (var i = 2; i < n; i++) {
    if (n % i === 0) return false;
  }
  return true;
}

// Print all primes up to N (e.g. N=10 → 2,3,5,7)
function printPrimes(n) {
  var result = [];
  for (var i = 2; i <= n; i++) {
    if (isPrime(i)) result.push(i);
  }
  console.log(result.join(", "));
}
printPrimes(10);   // 2, 3, 5, 7

Form Validation PYQ 7M

  
Validate before submitting — saves a server round-trip and gives instant feedback to users.

  
Mobile Number Validation PYQ 4M

  
function validateMobile() {
  var mob = document.getElementById("mobile").value;

  // Check 1: must be numeric only
  if (isNaN(mob)) {
    alert("Mobile number must contain digits only!");
    return false;
  }
  // Check 2: must be exactly 10 digits
  if (mob.length !== 10) {
    alert("Mobile number must be exactly 10 digits!");
    return false;
  }
  alert("Valid mobile number!");
  return true;
}

  
Full Registration Validation PYQ 7M

  
function validate() {
  var name = document.getElementById("name").value;
  var pass = document.getElementById("pass").value;
  var mob  = document.getElementById("mob").value;

  // All fields compulsory
  if (name === "" || pass === "" || mob === "") {
    alert("All fields are required!");
    return false;
  }
  // Password: 6–8 chars
  if (pass.length < 6 || pass.length > 8) {
    alert("Password must be 6 to 8 characters!");
    return false;
  }
  // Mobile: 10 digits
  if (isNaN(mob) || mob.length !== 10) {
    alert("Enter a valid 10-digit mobile number!");
    return false;
  }
  return true;  // allow submit
}

  
Email Validation using RegEx PYQ 7M

  
function validateEmail(email) {
  // Pattern: chars @ chars . chars
  var pattern = /^[\w\.-]+@[\w\.-]+\.\w+$/;
  var regex = new RegExp(pattern);
  return regex.test(email);
}

// Usage
function checkForm() {
  var email = document.getElementById("email").value;
  if (!validateEmail(email)) {
    alert("Invalid email address!");
    return false;
  }
  return true;
}

  
Password Strength Validation PYQ 7M

  
function strongPassword(pass) {
  // At least 8 chars, one capital, one number, one special char
  var hasUpper   = /[A-Z]/.test(pass);
  var hasNumber  = /[0-9]/.test(pass);
  var hasSpecial = /[!@#$%^&*]/.test(pass);
  var longEnough = pass.length >= 8;

  if (!longEnough) { alert("Min 8 characters"); return false; }
  if (!hasUpper)   { alert("Need 1 capital letter"); return false; }
  if (!hasNumber)  { alert("Need 1 number"); return false; }
  if (!hasSpecial) { alert("Need 1 special char"); return false; }
  return true;
}

Callbacks PYQ 3M PYQ 4M

  
A callback is a function passed as an argument to another function, to be called later. Used heavily for async operations.

  
// Basic callback
function process(callback) {
  console.log("Processing...");
  callback();   // call the passed function
}

process(function() {
  console.log("Done! Callback executed.");
});

// Real-world example: array methods

// sort() takes a callback comparator
nums.sort(function(a, b) { return a - b; });
console.log(nums);   // [1, 1, 3, 4, 5, 9]

// forEach takes a callback
nums.forEach(function(n) { console.log(n); });

// Callback with parameters
function doMath(a, b, operation) {
  return operation(a, b);
}
var result = doMath(10, 5, function(x, y) { return x + y; });
console.log(result);  // 15

JSON (JavaScript Object Notation) is a lightweight data format for storing and exchanging data. It's text-based, language-independent, and easy for both humans and machines to read.

  
Key Rule: JSON keys must be in double quotes. Values can be: string, number, boolean, array, object, or null.

    "state": "Gujarat"
  }

Error Handling PYQ 3M

  
Use try...catch...finally to handle runtime errors gracefully without crashing the page.

  
try {
  // code that might fail
  var result = riskyFunction();
  console.log(result);
}
catch(e) {
  // runs if error occurs in try block
  console.log("Error: " + e.message);
}
finally {
  // always runs, error or not
  console.log("Cleanup done.");
}

// Throw your own error
function divide(a, b) {
  if (b === 0) throw new Error("Cannot divide by zero!");
  return a / b;
}

try {
  console.log(divide(10, 0));
} catch(e) {
  console.log(e.message);   // Cannot divide by zero!
}
