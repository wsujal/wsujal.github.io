🏛️ What you already know — and why it matters

You know the 8085 has: ALU (does math/logic), Register Array (fast storage: A, B, C, D, E, H, L, SP, PC), CU (fetches & decodes instructions).

Assembly Language is just a human-readable way to write those instructions. Instead of binary 01000111, you write MOV B,A. The assembler converts it back. That's literally all assembly language is.

Every instruction tells the CU: "Put this data HERE" or "Do this math on A" or "Jump to that address if zero flag is set." Simple!

Core Concept #1
1-Byte, 2-Byte, 3-Byte Instructions
Every instruction in memory is 1 to 3 bytes. The first byte is ALWAYS the opcode. Click the examples below to visualize each instruction's bytes.

💡 The Mental Model

Think of it like a command + arguments. MOV B,A — the CPU knows from just the opcode which registers are involved, so no extra bytes needed. But MVI B, 32H — the opcode says "load B with data" but WHAT data? You need a 2nd byte. LDA 2050H — load accumulator from WHERE? A 16-bit address needs 2 more bytes (low byte first, then high byte).

1️⃣ 1-Byte Instructions

Opcode + Operand packed in one byte. The destination and source are implied by the opcode itself.

Examples: MOV B,A   LDAX D   PUSH B   RLC

Byte 1
Opcode

Memory footprint: 1 byte. Fastest to fetch.

2️⃣ 2-Byte Instructions

Opcode in byte 1, then an 8-bit immediate value or port address in byte 2.

Examples: MVI A, 32H   ADI 01H   IN 02   OUT 05

Byte 1
Opcode
Byte 2
8-bit Data

F+R = 2 machine cycles, 4+3 = 7 T-states.

3️⃣ 3-Byte Instructions

Opcode in byte 1, then a 16-bit address split into low byte (byte 2) and high byte (byte 3). 8085 is little-endian — low byte first!

Examples: LXI B, 2010H   JMP 3002H   CALL 4060H   LDA 1212H

Byte 1
Opcode
Byte 2
Addr Low
Byte 3
Addr High

Example: LDA 2050H → Byte1: LDA opcode, Byte2: 50H (low), Byte3: 20H (high)

Category 1
Data Transfer Instructions
Move data between registers, memory, and I/O ports. Key rule: Source data is NEVER modified. Flags are NOT affected.

💡 Quick Mental Map

All "load" instructions bring data into the accumulator or registers. All "store" instructions push data out. MOV is register-to-register. For memory, H&L pair acts as a pointer.

Pattern: MOV=reg-to-reg  |  MVI=immediate-to-reg  |  LDA/STA=direct memory (A only)  |  LXI=load 16-bit pair  |  LDAX/STAX=indirect via BC or DE

📚 PUSH & POP — Stack Operations Visualized

The stack grows downward in memory. SP (Stack Pointer) always points to the TOP of stack.

; Before PUSH B: SP = 2100H, B=50H, C=60H PUSH B ; Step 1: SP = SP-1 = 20FFH, M[20FFH] = B (50H) ; Step 2: SP = SP-1 = 20FEH, M[20FEH] = C (60H) ; After: SP = 20FEH POP B ; Step 1: C = M[SP=20FEH] = 60H, SP = SP+1 = 20FFH ; Step 2: B = M[SP=20FFH] = 50H, SP = SP+1 = 2100H ; After: SP = 2100H, BC restored!

Think of PUSH as "saving" registers and POP as "restoring" them — used around subroutines to preserve register values.

Category 2
Arithmetic Instructions
All arithmetic results go into the Accumulator (A). These instructions DO affect flags (S, Z, CY, AC, P).

💡 The One Rule: A is always the destination

ADD B means A = A + B. Not B = A + B. Not some third register. Always A.

For 16-bit addition: DAD B means HL = HL + BC. HL is the 16-bit accumulator equivalent.

The "I" suffix = Immediate: ADI 05H = A + 05H. The "C" suffix = with Carry: ADC B = A + B + CY flag (for multi-byte addition).

🔢 DAA — Decimal Adjust Accumulator

After adding two BCD numbers, the result might not be valid BCD. DAA fixes it automatically.

Rule: If lower nibble > 9 OR AC=1 → add 6 to lower nibble. If upper nibble > 9 OR CY=1 → add 6 to upper nibble.

; Add BCD 29 + 18 MVI A, 29H ; BCD 29 ADI 18H ; Binary result: A = 41H (but BCD should be 47!) DAA ; Lower nibble 1 < 9, upper nibble 4+1=5? No... ; Wait: 29H+18H = 41H. Lower = 1, no AC, upper = 4, no CY ; Hmm. Try: 29+18=47 decimal. DAA: 41H → add 06H → 47H ✓
Category 3
Logical Instructions
Bitwise operations and comparisons. All operate on the Accumulator. Results stored in A (except CMP which only sets flags).

💡 3 Patterns to Remember

Masking with AND: Use ANI 0FH to keep only lower nibble. AND with 1 = keeps bit. AND with 0 = clears bit.

Setting bits with OR: Use ORI 80H to set D7. OR with 1 = sets bit. OR with 0 = keeps bit.

Toggling with XOR: Use XRI FFH to flip all bits (same as CMA). XOR with 1 = flips. XOR with 0 = keeps.

ORA BA = A OR BS,Z,P mod; CY=0, AC=0

🔍 CMP vs CPI — Comparison Instructions

These are crucial for conditional jumps. They subtract but don't save the result — only set flags.

CMP B ; Compare A with B (A - B, flags only) ; if A < B → CY = 1 ; if A = B → Z = 1 ; if A > B → CY=0, Z=0 CPI 50H ; Compare A with 50H directly JZ EQUAL ; Jump if equal (Z=1) JC LESS ; Jump if A < 50H (CY=1)

CMP/CPI + Jump = if-else logic in 8085!

🎭 Practical Bit Manipulation Examples

; Check if a number is ODD (test bit D0) ANI 01H ; A AND 00000001 — isolates bit 0 JNZ ODD_LABEL ; if result ≠ 0, D0=1, number is odd ; Count 1s in a byte (used in Program 12) RAR ; rotate right: bit 0 goes to CY flag JNC SKIP ; if CY=0, bit was 0, skip counting INR B ; if CY=1, bit was 1, increment count
Category 4
Branch Instructions
Control program flow. JMP/CALL/RET with optional flag conditions. This is how 8085 implements if/else, loops, and functions.

💡 The Big Picture: Flags → Conditions → Jumps

Every conditional instruction has a letter code: C=Carry, NC=No Carry, Z=Zero, NZ=Not Zero, P=Positive, M=Minus, PE=Parity Even, PO=Parity Odd.

Same letter codes apply to J (Jump), C (Call), and R (Return). So JZ, CZ, RZ all check the same Z flag.

🔀 JMP — Unconditional Jump

Always jumps. PC ← address in instruction.

JMP 2050H ; PC = 2050H, next fetch from there ; Infinite loop: START: MVI A, 00H INR A JMP START ; loops forever

📞 CALL & RET — Subroutines

CALL addr: Pushes return address (next PC) onto stack, then jumps to addr.

RET: Pops return address from stack, jumps back.

CALL ADD1 ; save PC on stack, jump to ADD1 ... ; execution continues here after RET ADD1: ADD B ; subroutine body RET ; pop saved PC, return to caller

🔄 Loops in 8085 — The Standard Pattern

Loops use a counter register (usually C or B) + DCR + JNZ:

MVI C, 0AH ; count = 10 (0AH) LOOP: ; ... do something ... DCR C ; C = C - 1 (sets Z flag when C reaches 0) JNZ LOOP ; if C ≠ 0, repeat ; After loop: C = 00H, executed body 10 times

This is exactly how Programs 16, 17, 18, 19, 20 in Unit 5 work — they all use this pattern!

Core Concept #2
5 Addressing Modes
Every instruction needs to know where its data is. There are exactly 5 ways to specify this. Master these and instruction names become obvious.

💡 The "Where is the data?" Framework

When you write an instruction, the CPU needs to find the operand. It can be: baked into the instruction itself, at a specific memory address, in a register, pointed to by a register pair, or implied by the opcode.

1. Immediate
→
MVI B, 20H   LXI D, 1000H   ADI 05H
Data is IN the instruction bytes
2. Direct
→
LDA 2000H   STA 2050H   IN 08H
Address is IN the instruction bytes
3. Register
→
MOV A, B   ADD B   DCR C
Data is in a register
4. Indirect
→
LDAX D   STAX B   MOV A,M
Register pair holds the memory address
5. Implicit
→
RAL   XCHG   CMA   NOP
Operand is implied by opcode itself

Special Topic
Rotate Instructions
Shift bits left or right in the Accumulator. Critical for bit manipulation, serial I/O, and counting 1s in a byte.

💡 Two Types — With or Without Carry

Without Carry (RLC, RRC): The bit that falls off one end wraps around to the other end AND copies into CY.

Through Carry (RAL, RAR): The CY flag becomes part of the rotation — it acts as a 9th bit. Useful for multi-byte shifts.

Accumulator Value — Click a rotation to see it in action

RLC — Rotate Left (no carry)

D7 → D0 (wraps) and D7 → CY

; Before: A = A8H = 1010 1000, CY = ? RLC ; After: A = 51H = 0101 0001, CY = 1 ; D7(1) → CY, all bits shifted left, D7→D0

RRC — Rotate Right (no carry)

D0 → D7 (wraps) and D0 → CY

; Before: A = A8H = 1010 1000, CY = ? RRC ; After: A = 54H = 0101 0100, CY = 0 ; D0(0) → CY, all bits shifted right, D0→D7

RAL — Rotate Left Through CY

CY → D0, D7 → CY (9-bit rotation)

; Before: A = A8H, CY = 0 RAL ; After: A = 50H = 0101 0000, CY = 1 ; D7(1) → CY(new), old CY(0) → D0

RAR — Rotate Right Through CY

CY → D7, D0 → CY (9-bit rotation)

; Before: A = A8H, CY = 1 RAR ; After: A = D4H = 1101 0100, CY = 0 ; D0(0) → CY(new), old CY(1) → D7

🔬 Practical: Count 1s using RAR (Program 12)

MVI B, 00H ; B = count of 1 bits MVI C, 08H ; C = 8 (loop 8 times, for 8 bits) MOV A, D ; A = number to analyze BACK: RAR ; rotate right — bit 0 goes to CY flag JNC SKIP ; if CY=0, this bit was 0, skip INR B ; if CY=1, this bit was 1, count it SKIP: DCR C ; loop counter-- JNZ BACK ; repeat for all 8 bits HLT

Each RAR extracts the LSB into CY. The JNC checks it. Loop runs 8 times for 8 bits. Elegant!
